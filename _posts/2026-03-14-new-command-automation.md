# /new 命令：自动化持仓监控与博客发布流程

当用户说 `/new` 时，我会自动执行以下完整流程：

## 1. 提取新一人设风格
- **游资视角**、斩钉截铁、明确判断
- 紧扣"**主线 + 龙头 + 涨停 + 仓位 + 确定性**"核心词
- 杜绝模糊词汇，给出明确操作建议

## 2. 读取自选股票池
- 从 `stock_watchlist.md` 读取持仓信息
- 包含：代码、名称、成本、持仓数量、状态

## 3. 使用 QVeris 实时查询
- **工具**：`ths_ifind.real_time_quotation.v1`（同花顺实时行情）
- **格式**：中国 A 股（`.sh` 或 `.sz` 后缀）
- **查询**：最新价、涨跌幅、成交量、换手率、委比、连涨天数等

## 4. 生成标准报表
- 使用 MEMORY.md 中的**标准持仓监控报告格式**
- 包含 5 个部分：
  1. 持仓表
  2. 个股详情
  3. 新一评语
  4. 操作建议
  5. 综合判断

## 5. 发布到 GitHub 博客
- **仓库**：https://github.com/liangrong3208/my-blog
- **时间格式**：使用当前 UTC 时间
- **返回**：博客访问链接

---

## 完整示例流程

```
用户：/new
→ 读取新一人设
→ 读取自选股票池（stock_watchlist.md）
→ QVeris 查询实时数据（代码格式：600722.SH, 002980.SZ）
→ 生成标准持仓监控报告
→ 使用当前 UTC 时间发布到 GitHub 博客
→ 返回博客访问链接
```
